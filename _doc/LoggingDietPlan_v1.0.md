# Logging Diet & Code Cleanup Plan (v1.0)

## 1. 목적
- 코드 내 과도한 디버깅용 로그 라인을 제거하여 가독성 향상 및 바이너리 최적화.
- 필수적인 로깅 인프라(`CXLogDispatcher`, `CXFileLogger` 등)는 유지.
- 트레이딩 엔진 성능 개선을 위해 불필요한 IO 연산 최소화.

## 2. 기본 원칙
- **Essential Logging**: 시스템 오류, 주요 트레이딩 이벤트(진입/청산/수정), 치명적 예외 상황은 반드시 로깅.
- **Verbose/Trace Logging**: 단순 상태 추적, 반복적인 루프 내 출력, 디버깅 용도의 임시 로그는 모두 제거.

## 3. 작업 절차
1. **백업 (Mandatory)**: `git worktree`를 사용하여 현재 상태를 별도 브랜치로 백업 (`git worktree add ../ags-backup-logging-diet`). 오류 발생 시 즉시 롤백 가능 환경 조성.
2. **분석**: 전체 프로젝트에서 `Print`, `Log`, `Trace` 관련 키워드 검색 및 로그 패턴 식별.
3. **제거**: 불필요한 trace 레벨 로그 라인 일괄 제거.
4. **최적화**: 오류 발생 가능성이 높은 구간에만 엄격한 로깅 표준(GEMINI.md v11.1 준수)을 적용.
5. **검증**: 컴파일 및 정상 실행 확인, 실제 운영 중 필요한 로그가 누락되지 않았는지 확인.

## 4. 로깅 수준 분류
- **ERROR**: 시스템 동작 불가, 예외 상황. (유지)
- **INFO**: 주요 주문 처리, 상태 변경. (유지)
- **DEBUG/TRACE**: 로직 순서, 루프 내 상세 데이터. (전체 제거)

5. gitworktree를 사용하여 현재 버전 git에 백업하여 언제든지 rollbac 할 수 있는 작업 계획 수입
